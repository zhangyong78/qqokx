# OKX 策略工作台

当前版本：`v0.5.02`

这是一个面向 OKX 的桌面交易工作台，围绕“监控、交易、保护、回测、分析”构建。当前项目已经包含：

- 策略运行与多会话管理
- 账户持仓、历史成交、历史仓位
- 期权持仓保护与回放模拟
- 多币种信号监控与邮件通知
- Deribit 波动率指数查看与波动率监控
- 回测、矩阵对比、历史报告持久化
- 无限下单 / 条件单 / 网格任务
- 期权策略计算器与图表分析

## 1. 主界面与策略工作台

主界面支持同时运行多个策略会话，当前已接入：

- `EMA 动态委托-多头`
- `EMA 动态委托-空头`
- `EMA 穿越市价`
- `4H EMA5 / EMA8 金叉死叉`
- `现货增强三十六计`

主界面能力包括：

- 多策略同时运行
- 会话列表查看与停止
- 多 API 账户切换
- 策略会话与运行日志显示 `API配置`
- 账户持仓区优先显示
- 主界面不再显示选中持仓详情，重点聚焦策略与持仓
- 期权保护窗口会锁定当前选中的期权，不会被自动刷新改回第一条持仓

## 3. 期权持仓保护

支持针对已有期权仓位设置本地止盈止损保护。

支持方式：

- 按期权标记价格触发
- 按现货价格触发
- 设定价格报单
- 标记价格加减滑点报单

内置风控：

- 网络异常持续重试
- 3 分钟限频异常邮件
- 防重复平仓
- 反向仓位熔断
- 价格异常拦截
- 触发后锁定为待平仓，不回退到普通监控
- IOC 未成交撤单后自动进入平仓重试
- 多次未成交后进入 `持续未成交待人工`
- 成交日志与状态文案已统一为正常中文
- 保护任务列表、详情、日志与邮件会显示 `API配置`

价格拦截会参考：

- 现货价格
- 行权价
- 开仓均价
- bid / ask / last

如果报单价格明显异常：

- 不自动报单
- 任务进入异常
- 发邮件提醒
- 等待人工处理

平仓重试策略：

- 一旦止盈 / 止损触发，就进入 `已触发待平仓`
- 如果平仓单 `canceled` 且未成交，不直接记异常
- 系统会继续取最新盘口并重新尝试平仓
- 同一保护任务在相同未成交状态下不会重复发送邮件，避免撤单重报时刷屏
- 只有状态发生变化（如首次进入重试、转为 `持续未成交待人工`、最终成交、真正异常）才会再次发邮件
- 只有真正的业务错误、价格保护拦截或反向仓位等情况才会进入 `异常`
- 所有日志统一带时间前缀，当前采用更紧凑的 `[MM-DD HH:MM:SS]` 格式，便于排查成交与保护时序
- 任务详情区会记录触发标的、触发价格类型、止盈/止损报单方式、设定价格、滑点和轮询秒数，便于回看配置

## 5. Deribit 波动率工具

项目内包含两块 Deribit 波动率能力。

### 5.1 波动率指数窗口

支持：

- `BTC / ETH`
- `1小时 / 4小时 / 1天`
- 默认查看 `300` 根 K 线
- 波动率 K 线与同币种同周期现货 K 线双图联动显示
- 支持 `平均K线` 切换
- 鼠标滚轮缩放
- 左键拖动
- 双击重置视图
- 双图十字光标联动
- 页面打开后自动加载历史数据
- 切换币种 / 周期 / K线数量后自动刷新图表
- `日线对齐` 支持：
  - `北京时间凌晨12点`
  - `北京时间早上8点`

说明：

- Deribit 官方接口没有原生 `4小时 / 本地对齐日线`
- 程序会以 `1小时` 为底层数据源，本地聚合成：
  - `4小时`
  - `1天`
- `1天` 会先按 `4小时` 对齐，再聚合成日线
- 程序会从 `2021年` 开始建立波动率与现货 `1小时` 全量本地底仓
- 打开页面时会先读取本地缓存，再后台补最新小时数据
- 切换到 `4小时 / 1天` 时，不再重新整段拉接口，而是直接基于本地 `1小时` 缓存聚合
- 默认打开时只显示最近 `300` 根，但缩小视图或向左拖动时，会继续露出本地缓存中的更早历史
- `1小时` 周期最大支持 `30000` 根输入，界面默认仍显示最近窗口
- 自动刷新会按“最后一根K线的下一根边界”排程，而不是简单固定等待 `1` 小时
- 运行日志会带上当前已加载数据的 `最后一根K线` 时间，方便判断是否已经更新到最新
- `日线对齐` 选择会记住上次设置
- 网络超时时会优先保留当前已显示图表和缓存数据，不再用英文弹窗反复打断查看

本地缓存与存储：

- 波动率缓存文件：
  - `.okx_quant_deribit_volatility_cache.json`
- 存储位置：
  - `D:\qqokx\.okx_quant_deribit_volatility_cache.json`
- 文件性质：
  - 本地持久缓存文件
  - 不是临时文件
- 内容包括：
  - `BTC / ETH` 的全量 `1小时` 波动率 K 线底仓
  - 对应同币种现货全量 `1小时` K 线底仓
  - 最近获取时间
  - `日线对齐` 的界面记忆状态
- 典型缓存 key：
  - `BTC|hourly_base`
  - `ETH|hourly_base`
- 典型字段：
  - `volatility_candles`
  - `spot_candles`
  - `fetched_at`
  - `spot_inst_id`

## 6. 回测系统

入口：

- `工具 -> 打开回测窗口`
- `工具 -> 打开回测对比总览`

当前支持周期：

- `5分钟`
- `15分钟`
- `1小时`
- `4小时`

回测能力包括：

- 单次最多 `10000` 根已收盘 K 线
- 本地历史 K 线缓存
- 自动补最新历史数据
- 支持按时间段回测
- 自动补足指标预热和前置 K 线
- 前 `200` 根仅用于预热，不参与交易
- K 线图支持缩放、拖动、十字光标、悬停详情
- 资金曲线 / 净值相关展示
- 当前报告
- 回测对比
- 矩阵对比
- 参数热力图
- 周期统计
- 历史回测持久化

实盘策略运行时会自动按当前指标需要补足回看 K 线：

- `EMA 动态委托-多头`
- `EMA 动态委托-空头`
- `EMA 穿越市价`
- 本地触发版
- 只监控信号版

动态委托策略说明：

- `EMA 动态委托` 已拆成两套独立策略：
  - `EMA 动态委托-多头`
  - `EMA 动态委托-空头`
- 两套策略分别独立运行、独立回测、独立调参
- 每套都支持：
  - `固定止盈`
  - `动态止盈`
  - `每波最多开仓次数`
- 新增 `挂单参考EMA` 参数：
  - 用来决定挂单价格参考哪一条 EMA
  - 填 `0` 表示跟随 `EMA小周期`
  - 也可以单独填成 `5 / 8 / 13 / 21 / 34` 等周期，做成“信号 EMA”和“委托 EMA”分离
- `每波最多开仓次数` 会按趋势波段独立计数，趋势失效后重新形成时自动清零
- `动态止盈` 采用永久阶梯规则：
  - 达到 `2R`：止损移动到开仓价加双向手续费
  - 达到 `3R`：止损移动到 `2R`
  - 达到 `4R`：止损移动到 `3R`
  - 之后持续按 `nR -> 锁定 (n-1)R` 规则推进
- 动态委托回测与实盘都按“每根新 K 线撤旧挂新，使用最新 EMA 重新挂委托”的方式运行

回测结果增强：

- `交易明细` 支持显示：
  - `序号`
  - `止损值`
  - `ATR值`
  - `开仓数量`
  - `手续费`
- 回测结束时如果还有未平仓头寸，会在当前报告里单独列出 `期末未平仓`
- 回测报告会额外显示手续费拖累指标：
  - `手续费前盈亏`
  - `平均单笔手续费`
  - `手续费占手续费前盈亏`
  - `手续费占净盈亏绝对值`
  - `手续费占初始资金`

回测成本模型支持：

- Maker 手续费
- Taker 手续费
- 滑点
- 资金费率

ATR 批量矩阵规则：

- `止损 1 ATR -> 止盈 1 / 2 / 3 ATR`
- `止损 1.5 ATR -> 止盈 1.5 / 3 / 4.5 ATR`
- `止损 2 ATR -> 止盈 2 / 4 / 6 ATR`
- `固定止盈` 批量回测支持 `36` 组组合：
  - `SL x TP` 的 `3 x 3`
  - `每波最多开仓次数 = 0 / 1 / 2 / 3`
- `动态止盈` 批量回测支持 `12` 组组合：
  - `SL = 1 / 1.5 / 2 ATR`
  - `每波最多开仓次数 = 0 / 1 / 2 / 3`
- `矩阵对比` 与 `参数热力图` 会按当前止盈方式自动切换展示模式：
  - `固定止盈` 按开仓次数分层查看 `3 x 3`
  - `动态止盈` 按 `止损倍数 x 开仓次数` 显示 `3 x 4` 对比

## 7. 无限下单

入口：

- `工具 -> 打开无限下单`

这是一个单合约弹出式交易工作台。

特点：

- 一个窗口只服务一个合约
- 有活动任务时锁定当前合约
- 关闭窗口前必须停止任务并撤掉活动委托

### 7.1 手工下单

支持：

- 限价
- 最优价
- 市价（激进 IOC）
- IOC
- FOK
- Post Only

最优价逻辑：

- `buy` 挂 `买一`
- `sell` 挂 `卖一`

期权数量口径：

- 当前合约类型为 `OPTION` 时，数量按“币数”输入
- 程序会按合约面值自动换算成张数
- 仓位限制也按币数输入和显示，内部自动换算成张数

### 7.2 条件单

支持：

- 上穿 / 下穿触发
- 当前合约 / 自定义标的触发
- 触发后限价或激进 IOC

### 7.3 止盈止损

支持对当前合约单独启动本地止盈止损任务，并复用期权保护的严谨风控。

### 7.4 点击报价网格

这里不是传统铺多层静态网格，而是“点击报价后的往返挂单链”。

支持：

- 同一合约下多条网格任务并存
- 买成后按“多单参数”挂反向卖单
- 卖成后按“空单参数”挂反向买单
- 多单参数、空单参数可不同

### 7.5 规则盘口

支持：

- 显示交易所盘口买卖量
- 自定义价格筛选步长
- 当前价 / 买一 / 卖一高亮
- 点击价格带入
- 网格模式开启后，点击买卖列直接建网格任务

### 7.6 任务持久化

若软件异常关闭：

- 网格任务
- 条件单
- 止盈止损任务

会在下次打开时恢复显示，并允许再次启动。

## 8. 期权策略计算器

入口：

- `工具 -> 打开期权策略计算器`

支持：

- 读取期权链并显示认购 / 认沽报价
- 导入到期持仓
- 导入系列持仓
- 保存 / 加载 / 删除策略
- 组合公式线性表达式

- 组合 K 线
- 图表大窗里的 `波动率K线 / 叠加对比` 下图使用 `Deribit 波动率指数` 本地缓存

- 图表大窗

### 8.1 策略腿

当前策略腿表格支持：

- 持仓价
- 标记价
- Delta
- Gamma
- Theta
- Vega

其中：

- `持仓价` 可手工修改，用于分析真实建仓成本
- `标记价` 用于当前市值与现价参考
- 刷新报价不会覆盖持仓价

### 8.2 组合 K 线

默认值：

- `K线周期 = 1H`
- `K线数量 = 1000`
- `图表币种 = 结算币`

特点：

- 组合 K 线采用 OKX 期权标记价格组合
- 支持缩放、拖动、十字光标
- 支持 `只影响组合K线`
- 支持 `消除影线`
  - 开启后只按 `open / close` 计算高低
- 颜色规则为：
  - `绿涨`
  - `红跌`

### 8.3 图表大窗

图表大窗包含：

- 到期盈亏图
- 组合K线
- 波动率K线
- 叠加对比

特点：

- 到期盈亏图支持 hover 跟随线
- 组合K线支持缩放、拖动、十字光标
- 波动率K线周期与组合K线一致
- `叠加对比` 改为上下双图
  - 上图：组合K线
  - 下图：波动率K线
- 上下双图共用时间轴
- 十字光标上下图联动

## 9. 期权展期建议

入口：

- 主界面持仓区选中单腿卖出期权后点击 `展期建议`
- 持仓大窗中选中单腿卖出期权后点击 `展期建议`

第一版定位：

- 面向 `short call / short put`
- 自动扫描后续到期可用于展期/移仓的候选合约
- 给出 `平旧 + 开新` 的建议排序
- 可一键 `送入期权策略计算器`

当前支持：

- 扫描条件：
  - `目标偏好`
  - `行权价方向`
  - `档位优先`
  - `候选数量`
- 行权价方向支持：
  - `更虚值优先`
  - `同行权价优先`
  - `同行权价及更安全方向`
  - `全部`
- 目标偏好支持：
  - `优先净收权利金`
  - `优先降低风险`
  - `优先保持 Delta 接近`
  - `优先时间价值`
  - `优先更近到期`
- 建议结果直接显示：
  - `买一价`
  - `卖一价`
  - `标记价`
  - `预计净收/付`
  - `新Delta`
  - `风险变化`
  - `价差`
- 没有 `买一价` 的候选合约会自动过滤，不纳入建议结果

- `.okx_quant_credentials.json`
- `.okx_quant_settings.json`

- `.okx_quant_backtest_history.json`

更新说明口径：

- `patch`：文档、提示文案、UI 小问题、测试和 bug 修复
- `minor`：新增明显功能、新窗口、新面板或新工作流

- 如果同版同时有小修和大功能，按用户可感知的最大变化判断版本级别

## 数据目录与恢复接管

本轮已把运行期数据从代码目录中抽离，默认改为使用项目同级的共享数据目录 `qqokx_data/`。

启动方式：
- 默认直接运行 `python main.py`，会自动使用代码目录同级的 `qqokx_data/`
- 也可以显式指定：`python main.py --data-dir D:\qqokx_data`
- 也支持环境变量：`QQOKX_DATA_DIR=D:\qqokx_data`

数据目录结构：
- `config/`：API 凭证、通知设置、增强策略运行配置
- `cache/`：本地 K 线缓存、Deribit 波动率缓存
- `state/`：回测历史、策略历史、Smart Order 任务、可恢复策略会话清单
- `logs/`：全局日志与策略独立日志
- `reports/`：CSV / TXT / 分析报告 / 实盘会话导出

兼容迁移：
- 首次启动会自动把旧版 `.okx_quant_*`、`logs/`、`reports/` 迁移到新数据目录
- 升级时推荐“只替换代码目录，保留原 `qqokx_data/`”

恢复接管：
- 现货增强三十六策略已支持“保守恢复”
- 应用关闭后会保留会话快照、事件流和运行目录
- 下次启动后，主界面会把对应会话标记为“待恢复”，并尝试自动接管；也可以手动点“恢复选中策略”
- 恢复时会先读取本地快照，再对账账户已有仓位和成交，避免重复识别旧信号或旧成交
- 普通策略当前仍不承诺自动接管；Smart Order 继续沿用已有持久化恢复链路
