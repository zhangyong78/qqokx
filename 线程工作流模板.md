# 线程工作流模板

这份文档用于给 `qqokx` 项目长期分线程协作时直接复用，目标是：

- 让不同功能的上下文保持干净
- 让历史记录更容易回查
- 让每条线程都有清晰边界
- 避免把 UI、回测、实盘、发布混在一起

## 1. 推荐长期线程结构

建议固定使用以下 `8` 个长期线程：

1. `总控-版本路线图`
2. `回测-策略逻辑`
3. `回测-图表与报告UI`
4. `实盘-策略执行与下单`
5. `主界面-持仓与历史`
6. `期权-保护与策略工具`
7. `监控-信号与波动率`
8. `数据-缓存-导出-发布`

## 2. 每个线程负责什么

### 2.1 总控-版本路线图

用途：

- 记录当前版本目标
- 排优先级
- 跟踪哪些功能在哪个线程继续
- 做阶段总结

不放：

- 具体代码细节
- 某个功能的小修小补

开场模板：

```text
这个线程只处理项目总控、版本规划、优先级，不直接展开具体代码细节。

线程约定：
- 我说 `123` 时，表示按仓库里的发版协作约定执行本线程对应动作。

当前范围：
- 版本目标
- 当前优先级
- 各功能线程分工
- 阶段总结
- 发布节奏

不处理：
- 具体代码实现
- 细节级调参
- 单个功能的小修小补

当前目标：
- 明确当前版本要做什么
- 明确每个功能在哪个线程继续
- 阶段结束时沉淀总结
```

### 2.2 回测-策略逻辑

用途：

- 处理回测里的策略逻辑
- 处理止盈止损和开仓规则
- 处理手续费、滑点、资金费等统计口径

不放：

- 回测窗口布局
- 主界面 UI

开场模板：

```text
这个线程只处理回测策略逻辑，不改主界面UI，不处理打包发布。

线程约定：
- 我说 `123` 时，表示把本线程已完成内容登记到 `发版待打包清单.md`。

当前范围：
- EMA 动态委托回测
- 固定止盈 / 动态止盈
- 每波最多开仓次数
- 手续费 / 滑点 / 资金费
- 最小下单量兜底
- 回测统计口径

不处理：
- 回测窗口布局
- 主界面显示
- 打包和版本发布

当前目标：
- 保证回测逻辑正确
- 保证报告口径一致
- 让策略实验可复现
```

## 3. 哪些内容不要硬拆

以下内容强相关，建议放在同一个线程里持续处理：

- `回测策略逻辑 + 回测报告口径`
- `实盘策略 + API配置 + 下单`
- `Deribit波动率图表 + 波动率监控`
- `期权保护 + 保护日志 + 保护邮件`

## 4. 什么时候新开线程

适合新开线程的情况：

- 一个大功能准备长期迭代
- 当前线程已经混入太多不相关内容
- 你准备把一个阶段和下一个阶段明确分开
- 同一功能出现了明显的新方向

不需要新开的情况：

- 只是同一功能的小修补
- 明天还会继续沿着原方向改
- 当前线程上下文仍然很清晰

## 5. 什么时候归档

适合归档：

- 这个功能阶段已经做完
- 这个问题已经修掉
- 这个线程短期大概率不会再继续
- 你准备换一个更清晰的新阶段线程

先不要归档：

- 这个功能还在持续迭代
- 第二天还要继续改
- 线程里还有很多重要上下文没有沉淀

## 6. 推荐实际工作流

建议长期这样使用：

1. 永久保留一个 `总控-版本路线图`
2. 其他线程按功能长期存在
3. 小改动继续原线程
4. 大方向变化再新开线程
5. 功能做完后再归档
6. 每个线程结束前，让 Codex 帮你写一段“当前状态总结”

### 6.1 发版交接补充规则

为了避免发版依赖某一个聊天窗口记住口头约定，建议再固定一条：

- 功能线程完成一批可发版改动后，让 Codex 把结果登记到 `发版待打包清单.md`
- `总控-版本路线图` 或 `数据-缓存-导出-发布` 线程统一读取 `发版待打包清单.md`
- 发版线程负责：
  - 更新版本号
  - 更新 `README.md` 更新日志
  - 打包
  - 提交
  - 推送

推荐口令约定：

- 功能线程里说 `123`
  - 默认表示：登记当前线程已经完成、准备进入下一版的内容
- 总控或发布线程里说 `123`
  - 默认表示：消费 `发版待打包清单.md`，统一完成发版动作

这样做的重点是：

- 线程负责产出
- 总控负责收口
- 约定写进仓库，而不是只留在聊天记忆里

## 9. 本项目当前建议

对 `qqokx` 这个项目，最推荐的做法就是：

- 项目方向、版本安排，放 `总控-版本路线图`
- 策略规则变化，放 `回测-策略逻辑`
- 回测页面、矩阵、热力图，放 `回测-图表与报告UI`
- 实盘执行、委托刷新、API，下放 `实盘-策略执行与下单`
- 主界面持仓、历史成交、历史仓位，放 `主界面-持仓与历史`
- 期权保护、策略计算器、展期建议，放 `期权-保护与策略工具`
- 信号监控、波动率指数、联动图表，放 `监控-信号与波动率`
- 缓存、导出、README、开发指南、打包发布，放 `数据-缓存-导出-发布`

这样做的核心好处是：

- 查历史更快
- 上下文更干净
- 功能边界更稳定
- 后期项目继续变大时也不容易乱
