# 线程工作流模板

这份模板用于长期协作拆线，目标是让不同线程各做各的，最后由总控线程统一收口发版。

## 1. 推荐的线程结构

建议固定使用以下 `8` 类线程：

1. `总控-版本路线图`
2. `回测-策略逻辑`
3. `回测-图表与报告UI`
4. `实盘-策略执行与下单`
5. `主界面-持仓与历史`
6. `期权-保护与策略工具`
7. `监控-信号与波动率`
8. `数据-缓存-导出-发版`

## 2. 每个线程负责什么

### 2.1 `总控-版本路线图`

用途：

- 记录当前版本目标
- 排优先级
- 跟踪哪些功能在哪个线程继续
- 做阶段总结

不处理：

- 具体代码实现
- 细节级调参
- 单个功能的小修小补

### 2.2 `回测-策略逻辑`

用途：

- 处理回测里的策略逻辑
- 处理止盈止损和开仓规则
- 处理手续费、滑点、资金费等统计口径

不处理：

- 回测窗口布局
- 主界面 UI

### 2.3 `回测-图表与报告UI`

用途：

- 处理回测窗口表现层
- 处理回测图表与报告展示
- 处理交易明细、矩阵对比、热力图

不处理：

- 回测核心交易逻辑
- 实盘执行链路

### 2.4 `实盘-策略执行与下单`

用途：

- 处理实盘策略执行
- 处理下单、撤单、挂单刷新
- 处理 API 配置与订单执行问题

不处理：

- 回测 UI
- 主界面持仓展示细节

### 2.5 `主界面-持仓与历史`

用途：

- 处理主界面布局
- 处理账户持仓、持仓大窗
- 处理历史成交和历史仓位

不处理：

- 回测逻辑
- 实盘下单核心逻辑

### 2.6 `期权-保护与策略工具`

用途：

- 处理期权保护
- 处理期权策略计算器
- 处理图表大窗与展期建议

不处理：

- 回测主系统
- 主界面公共表格

### 2.7 `监控-信号与波动率`

用途：

- 处理信号监控
- 处理波动率监控
- 处理 Deribit 波动率指数图表与刷新

不处理：

- 回测系统
- 实盘下单细节

### 2.8 `数据-缓存-导出-发版`

用途：

- 处理本地缓存
- 处理导出文件
- 处理文档、打包、提交和发版

不处理：

- 具体策略逻辑设计
- 主界面视觉微调

## 3. 每个线程结束时做什么

每个线程只登记已经完成、准备进入下一版的结果，不写具体实现细节。

建议登记：

- 线程名
- 模块或功能名
- 面向用户的变化摘要
- 版本建议：`patch` / `minor` / `major`
- 备注：是否暂时只登记、不发版

## 4. 总控线程的发版规则

- 功能线程说 `123`，默认表示这一轮内容可以进入发版清单
- 总控线程说 `123`，默认表示执行完整发版流程
- 完整发版流程包括：
  - 更新版本号
  - 更新中文更新日志
  - 打包
  - 提交
  - 推送

## 5. 一次性启用模板

如果某个线程在这套模板之前就已经存在，可以直接补一句：

```text
本线程启用 123 约定：
我说 123 时，只表示把本线程已经完成的内容登记到发版清单，
由总控线程统一决定是否进入下一版发版流程。
```

这样后续线程之间的边界会更清楚。
