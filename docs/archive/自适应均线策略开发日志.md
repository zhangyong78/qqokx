# 自适应均线策略开发日志

日期：2026-05-13

## 一、策略理解

这套策略的核心，不是传统的 `EMA 金叉/死叉`，而是识别：

当前市场到底正在尊重哪一条 `EMA`，并把它当成推进轨道来运行。

策略目标不是预测，而是跟随已经形成的结构，围绕“被市场尊重的 EMA 轨道”反复交易，直到这条轨道第一次真正失效。

这意味着策略的关键，不在于均线本身，而在于下面几个定义是否严格：

- 什么叫“触碰轨道”
- 什么叫“回踩成功”
- 什么叫“假跌破”
- 什么叫“轨道失效”
- 什么叫“主导 EMA 切换”

如果这些定义不清晰，回测结果就会高度依赖主观解释，策略很容易出现“图上看着对，程序里做不稳”的问题。

---

## 二、总体判断

这个策略值得做，而且适合接入当前项目。

原因：

- 仓库里已经有成熟的 `EMA` / `ATR` 指标计算能力
- 已有回测主链路、策略目录、参数模型、风控骨架
- 现有结构支持新增独立策略，不需要从零搭系统

但这类策略最容易出问题的地方，不是代码实现，而是规则定义：

- 轨道识别是否会抖动
- 评分是否事后偏置
- 趋势过滤是否足够严格
- 同一段行情是否被重复计数

所以第一版最重要的目标不是“做复杂”，而是“做清楚”。

## 五、Respect Score 建议

第一版建议使用可解释的规则评分，不要直接上机器学习。

建议先做线性评分：

```python
score =
    touch_score
    + bounce_score
    + slope_score
    - fake_break_penalty
    - break_penalty
```

推荐评分维度：

1. 有效触碰次数
2. 回踩反弹成功率
3. EMA 斜率强度
4. 假跌破次数
5. 有效跌破次数

这样做的优势：

- 结果可解释
- 便于调参
- 回测时能知道到底哪项在贡献收益
- 后续替换为数据驱动权重也容易

## 六、最关键的定义建议

这部分是整个策略成败的核心。

### 1. touch 定义

建议不要用“价格接近 EMA”这种模糊说法。

建议定义为：

```python
abs(price - ema) <= k * ATR
```

例如：

```python
k = 0.2
```

也就是价格进入 `EMA ± 0.2 ATR` 区域，才算一次触碰。

### 2. bounce success 定义

建议定义为：

- 价格触碰 EMA 后
- 在后续 `N` 根 K 线内
- 至少走出 `Y * ATR` 的反弹幅度
- 且期间不能先发生有效跌破

这样可以避免把“碰一下就继续横盘”误判成成功支撑。

### 3. fake break 定义

建议定义为：

- 盘中跌破 EMA
- 但收盘重新站回
- 或下一根很快收回

这类情况不能直接算轨道失效，更适合作为“假突破/假跌破”计入惩罚或弱化项。

### 4. broken 定义

建议轨道失效不要只看单根影线。

建议优先用：

- 收盘有效跌破 `EMA - X * ATR`
- 连续若干根无法收回
- EMA slope 走平或转弱
- 高低点结构破坏

### 5. rail switch 定义

这是最容易抖动的地方。

建议不要出现 `EMA21` 和 `EMA34` 来回切换。

可以增加切换门槛：

- 新轨道分数必须高出旧轨道一个阈值
- 连续若干根都保持优势
- 或旧轨道先进入 `WEAKENING`

## 七、交易逻辑建议

### 1. 方向

第一版只做多，合理。

原因：

- `BTC` 长期偏多头
- 多头结构通常比空头更稳定
- 有利于先验证策略主干逻辑

### 2. 入场

文档里“前两次观察，第三次开始挂单”的思路很好，建议保留。

可以理解为：

- 前两次只用于确认该 EMA 的确被市场尊重
- 第三次开始参与，避免太早进场

建议入场区：

```python
entry_zone = EMA ± 0.2 ATR
```

### 3. 止损

建议沿用 ATR 动态止损，方向正确。

例如：

```python
stop = EMA - 1.2 ATR
```

但需要注意：

- 止损距离不能太近，否则轨道类策略会被正常回踩频繁洗掉
- 止损距离不能只凭经验，必须做矩阵回测

### 4. 止盈

不建议固定止盈，这一点认同。

轨道策略的优势本来就来自“让利润沿结构延伸”，固定止盈会切断大段趋势收益。

第一版建议：

- 以轨道失效作为主退出逻辑
- 必要时叠加保本上移机制

## 九、回测重点

这类策略回测不能只看胜率。

更重要的是看：

- 盈亏比
- 最大回撤
- 趋势行情贡献度
- 震荡行情损耗
- 连续亏损次数
- 不同市场阶段的表现分布

特别建议新增两类分析：

### 1. 轨道稳定性分析

记录：

- 主导 EMA 切换频率
- 单条轨道平均持续时长
- 每条轨道的平均触碰次数

如果轨道切换过于频繁，说明评分体系不稳定。

### 2. 信号质量分析

记录：

- 第 1 次、第 2 次、第 3 次回踩的胜率差异
- 不同 EMA 周期的胜率和盈亏比
- 不同 `ATR` 宽度下的表现

这样才能知道：

是“所有轨道都有效”，还是“只有某几类轨道有效”。

## 十八、市场状态过滤方案

V1 建议不要一口气上太多过滤器。

推荐第一版只做三项：

1. `价格收盘在 EMA200 上方`
2. `EMA200 slope > 0`
3. `近 N 根高点和低点整体抬高`

可以先不接 `ADX`。

原因：

- 你当前指标层里还没有现成 `ADX`
- 先用简单趋势判定，可以更快进入验证阶段
- 后续如果震荡过滤不够，再补 `ADX` 或波动过滤

建议输出：

```python

RANGE
UNQUALIFIED
```

V1 不需要把 `TREND_DOWN` 和 `VOLATILE` 做得很复杂，因为当前只做多。

### 建议判定口径

```python
trend_ok =
    close > ema200
    and ema200_now > ema200_n_bars_ago
    and recent_swing_highs_lows_rising
```

其中：

- `ema200_now > ema200_n_bars_ago` 用于判断大方向抬升
- `recent_swing_highs_lows_rising` 用于避免 EMA 刚抬头但结构还没站稳

建议每根新 K 线都对候选 EMA 重新评分，但不要每次都立刻切换主轨道。

```python
lookback = 60 bars
```

### 对每条 EMA 计算的中间量

- `bounce_rate`
- `fake_break_count`
- `hard_break_count`
- `slope_strength`

### slope_strength 建议

不要直接用价格差，而要归一化到 ATR。

```python

```

优点：

- 不同价格区间可比
- 后续用于分数加权更稳定

## 二十、Respect Score 初版公式

建议第一版用 100 分制，保持可读性。

### 评分公式

```python
respect_score =
    touch_score
    + bounce_score
    + slope_score
    + recency_score
    - fake_break_penalty
    - hard_break_penalty
```

### 推荐拆分

```python
touch_score = min(touch_count, 5) * 8
bounce_score = min(bounce_count, 4) * 12
slope_score = clamp(slope_strength, 0, 2) * 10
recency_score = max(0, 10 - last_touch_age)
fake_break_penalty = fake_break_count * 6
hard_break_penalty = hard_break_count * 12
```

### 评分解释

- `touch_score` 代表市场是否多次和该 EMA 互动
- `bounce_score` 代表这些互动是否真的产生了支撑效果
- `slope_score` 代表轨道是否具有推进方向
- `recency_score` 防止系统长期抱着一条“老轨道”不放
- `fake_break_penalty` 和 `hard_break_penalty` 用来打击脆弱轨道

### V1 主轨道选择规则

```python
best_ema = argmax(respect_score)
```

但只有在以下条件同时满足时才认定为候选主轨道：

- `touch_count >= rail_min_touches`
- `bounce_count >= rail_min_bounces`
- `slope_strength > 0`
- `respect_score >= 55`

## 二十一、touch / bounce / break 的程序定义

这一节必须写严，否则代码和肉眼会对不上。

### 1. touch

建议定义：

```python
abs(low_or_close - ema_value) <= rail_touch_atr_ratio * atr_value
```

对于多头支撑轨道，更推荐使用：

```python
low <= ema + zone and close >= ema - zone
```

这样既承认价格回踩过，又避免把完全跌穿后的远离位置误当成 touch。

### 2. bounce success

建议定义：

- 发生 touch 后
- 后续 `rail_bounce_confirm_bars` 根内
- 最高价相对 touch bar 至少上行 `rail_bounce_atr_ratio * ATR`
- 且期间没有先发生 hard break

### 3. fake break

建议定义：

- 盘中跌到 `EMA - touch_zone` 下方
- 但收盘重新站回 `EMA`
- 或下一根 K 线重新站回 `EMA`

### 4. hard break

建议定义：

```python
close < ema - rail_break_atr_ratio * atr
```

或者：

- 连续 `rail_reclaim_bars` 根都未能收回 EMA

V1 可以把两种都纳入 hard break。

## 二十二、状态机落地建议

文档里原始状态很多，理念很好，但第一版代码建议先简化成 5 个运行态。

### 建议的运行态

```python
SEARCHING
CANDIDATE
CONFIRMED
ACTIVE
BROKEN
```

### 状态说明

`SEARCHING`

- 正在趋势市场里寻找可能的主导 EMA

`CANDIDATE`

- 已找到当前最高分 EMA
- 但还没达到最小确认条件

`CONFIRMED`

- 已满足触碰、反弹、方向一致等条件
- 允许等待下一次回踩入场

`ACTIVE`

- 已存在该轨道上的交易行为
- 轨道继续有效时允许反复参与

`BROKEN`

- 当前轨道被确认破坏
- 进入冷却和重新寻找阶段

### 建议状态转移

```python
SEARCHING -> CANDIDATE
当某条 EMA 分数最高且超过最低阈值

CANDIDATE -> CONFIRMED
当该 EMA 满足最少 touches / bounces / slope

CONFIRMED -> ACTIVE
当价格再次进入 entry zone 并触发交易

ACTIVE -> BROKEN
当发生 hard break 或趋势过滤失效

BROKEN -> SEARCHING
经过冷却 bars 后重新评估
```

### 为什么要简化

因为第一版真正影响收益的不是状态名字，而是：

- 什么时候允许交易
- 什么时候禁止交易
- 什么时候认定轨道废了

这些明确了，之后再映射回更细的展示状态也不晚。

## 二十三、V1 入场与出场方案

### 入场

建议沿用你文档里的“前两次确认、第三次参与”。

可程序化为：

- 当前主轨道已 `CONFIRMED`
- 历史上已出现至少 `2` 次成功支撑

## 二十九、开发风险提醒

这套策略最容易踩的坑有四个。

### 1. 未来函数

`bounce success` 的确认一定不能偷看未来太多 bars。

解决建议：

- 识别层和交易层严格区分
- 回测里只允许使用当时已经可见的信息

### 2. 轨道切换抖动

如果没有切换阈值，`EMA21` 和 `EMA34` 很容易来回抢主导地位。

解决建议：

- 引入 `rail_switch_min_score_delta`
- 引入最短持有轨道 bars

### 3. 评分过拟合

如果参数太多，回测很好看，但出样本会塌。

解决建议：

- V1 限制可调参数数量
- 先做少量默认值回测，不急着扫参

### 4. 震荡误判

趋势过滤不够时，轨道策略在横盘里会被来回打脸。

解决建议：

- 先用严格的 `EMA200 + 结构抬高` 过滤
- 后续再决定是否补 `ADX`

研究口径：

- 标的：`BTC-USDT-SWAP`
- 周期：`1H` / `4H`
- 数据：本地 `candle_cache` 全量已确认 K 线

- 时间窗：`全历史`、`自 2024-01-01`、`自 2025-01-01`

### 核心结论

#### 1. 这个方向不是没价值，但价值不在 1H

当前第一版 `Adaptive Rail-1H` 在真实 BTC 历史回测中，没有形成可观优势。

代表结果：

- 全历史：收益约 `0.97%`，最大回撤约 `1.82%`
- 2024+：收益约 `-0.28%`
- 2025+：收益约 `-1.05%`

对比现有基线：

- `EMA 动态委托-标准1H` 全历史收益约 `9.47%`
- `EMA 突破做多-1H` 全历史收益约 `6.11%`

这说明：

当前这版轨道定义放到 `1H` 上，噪声影响太大，还没有跑出稳定优势。

#### 2. 4H 版本有继续研究的价值

`Adaptive Rail-4H` 的表现明显更像一个成立的研究方向。

代表结果：

- 全历史：收益约 `4.85%`，最大回撤约 `1.03%`
- 2024+：收益约 `1.77%`，最大回撤约 `0.74%`
- 2025+：收益约 `0.96%`，最大回撤约 `0.75%`

它的特点不是“收益第一”，而是：

- 回撤低
- 盈亏质量稳定
- 在近两年和近一年样本里没有失真得很厉害

#### 3. 4H 上它更像“低频结构跟随”，不是“高频增强器”

从研究结果看，这个方向更适合：

- 更慢的周期
- 更干净的趋势结构
- 更强调回撤控制

而不适合：

- 在 `1H` 上追求高频交易机会
- 靠频繁切换轨道去榨取小波动

#### 4. 当前最合理的判断

可以把这个方向定义为：

```text
Adaptive EMA Rail 有研究价值，
但当前价值主要集中在 4H 低频趋势结构版本，
而不是 1H 通用版本。
```

### 与基线策略的相对位置

当前结论更适合这样理解：

- 如果目标是 `1H 收益效率`，现有 EMA 基线策略仍更强
- 如果目标是 `4H 更低回撤的结构型做多策略`，Adaptive Rail 值得继续打磨

它不是现阶段的“主策略替代者”，更像一个：

```text
有潜力成为 4H 风险更稳的补充型趋势策略
```

### 后续研究建议

基于这轮结果，后续不建议平均用力。

建议明确收口：

1. 暂停深挖 `1H Adaptive Rail`
2. 集中优化 `4H Adaptive Rail`
3. 重点研究：
   - 轨道切换阈值
   - bounce 成功定义
   - broken 判定
   - 单条轨道最大开仓次数
   - 轨道持续时长与切换频率统计

### 当前一句话判断

这个方向：

```text
值得继续研究，但只建议把资源继续投到 4H 版本。
```


也就是说，它的弱点更偏向：
- 失效定义
- 止损宽度
- 失效退出与动态止盈的配合

而不是主导 EMA 选择本身。

## 四十、EMA21 适用的市场结构

这轮继续深挖的问题是：

```text
EMA21 在什么市场结构下会重新变得有用？
```

- 价格离 `EMA200` 的距离（按 ATR 标准化）
- `EMA55` 斜率
- `EMA200` 斜率
- `EMA21` 与 `EMA55` 的张口程度
- 最近 `8` 根 K 线振幅
- `ATR100` 分位

研究脚本：
```text
scripts/run_adaptive_ema21_market_structure_study.py
```

研究报告：


### 核心结论

#### 1. EMA21 更偏爱“强趋势已展开”的结构，不是普通慢趋势

对比 `2024+` 样本里 `EMA21` 与非 `EMA21` 交易：

- `EMA21` 交易的价格离 `EMA200` 更远：`6.69 ATR` vs `3.62 ATR`
- `EMA55` 斜率更陡：`1.12` vs `0.41`
- `EMA200` 斜率也更强：`0.75` vs `0.46`
- `EMA21-EMA55` 张口更大：`1.99 ATR` vs `0.80 ATR`

这说明：

```text
EMA21 有用的时候，
往往不是“趋势刚开始”，
而是“趋势已经明显展开，而且快慢均线已经拉开层次”。
```

它更像在抓：
- 强势多头推进中的快轨道回踩
- 中段加速后的再次承接

而不是一般性的平缓上行。

#### 2. EMA21 不是在“波动越大越好”的地方有效

很多人会自然猜它只适合高波动爆发行情，但这轮结果更细：

- `EMA21` 交易的 `ATR100` 分位确实偏高：`62.60%`
- 但最近 `8` 根 K 线振幅反而没有更大：`2.46 ATR`，低于非 `EMA21` 的 `3.01 ATR`

这说明它更像：

```text
长期波动和趋势强度已经被抬高，
但短期局部结构不能太失控
```

也就是：
- 可以是强趋势
- 可以有偏高 ATR
- 但不适合刚经历极端短线爆冲后马上去追

#### 3. EMA21 的大亏单，往往出现在“过热追击”

看 `2024+` 的几笔 `EMA21` 交易：

- `2024-03-05` 是最大亏单，约 `-1R`
- 当时的 `ATR100` 分位是 `100%`
- 最近 `8` 根 K 线振幅高达 `4.50 ATR`
- 但价格离 `EMA200` 的距离只有 `3.43 ATR`
- `EMA21-EMA55` 张口也只有 `1.42 ATR`

这很像一种典型坏结构：

```text
短线波动已经极热，
但中层趋势展开程度还不够扎实，
于是 EMA21 回踩更像追击噪声，而不是顺着成熟快轨道承接
```

#### 4. EMA21 的大赚单，更像“强趋势里的有序回踩”

两笔最明显的 `EMA21` 盈利单：

- `2024-03-08`：约 `+2R`
- `2024-11-17`：约 `+3R`

它们的共同点更接近：

- 价格离 `EMA200` 很远：`7.63 / 8.99 ATR`
- `EMA55` 与 `EMA200` 斜率都很强
- `EMA21-EMA55` 张口明显：`1.61 / 2.57 ATR`
- 最近 `8` 根振幅不算夸张：`2.44 / 1.54 ATR`

这说明真正适合 `EMA21` 的结构更像：

```text
主趋势已经拉开，
快轨道明显领先中轨，
但短线局部回踩仍然有序，没有进入失控爆冲
```

#### 5. 为什么 2025+ 里 EMA21 完全没用

现在也更容易解释 `2025+` 为什么没有任何 `EMA21` 交易了：

- 价格离 `EMA200` 的平均距离只有约 `3.70 ATR`
- `EMA55` 斜率约 `0.51`
- `EMA21-EMA55` 张口约 `0.97 ATR`

这说明 `2025+` 这段 4H 结构更像：
- 趋势在走
- 但没有形成很多“快轨道大幅领先中轨”的结构
- 所以系统自然把主导轨道留给了 `34/55/89`

也就是说，不是 `EMA21` 失效了，而是：

```text
2025+ 这段样本，本来就不太提供 EMA21 最擅长的那种快轨道结构
```

### 现在可以落成一句策略判断

## 四十四、这个策略准备怎么用

到现在为止，这个策略的定位已经不适合再写成“看看能不能做出一个更高收益的 EMA 版”。

更准确的用法应该是：

```text
把它当成 4H 低回撤、重交易质量的结构型做多策略
```

而不是：

```text
把它当成全场景主收益引擎
```

### 我准备这样用它

#### 1. 作为 `BTC 4H` 的质量型主研究策略

当前最自然的第一用途是：
- 只做 `BTC`
- 只做 `4H`
- 只做 `LONG`
- 只跑默认底座：
  `Balanced 4H + 21/34/55/89 + EMA21 gate`

原因很简单：
- 这是目前已经被数据验证过的最稳口径
- 回撤、PF、平均R 的形态最好
- 不需要再拿 `1H` 去强行扩场景

#### 2. 作为“低回撤趋势腿”，不是“高频赚钱腿”

如果后面要和现有策略一起用，我不会把它和 `EMA Dynamic 5/13 4H` 视为同一角色。

更合理的分工是：

- `EMA Dynamic 5/13 4H`
  偏收益、偏积极、交易更多
- `EMA Breakout 4H`
  偏直接、偏突破跟随
- `Adaptive Rail 4H`
  偏结构质量、偏低回撤、偏克制

所以它最适合承担的是：

```text
组合里“更稳的那条趋势腿”
```

而不是全仓单押去追求收益第一。

#### 3. 作为“允许开仓的结构过滤器”

如果后面你不想单独把它当一套完整策略跑，
它还有第二种很实用的用法：

```text
把 Adaptive Rail 的“已确认主导轨道”当成结构过滤器
```

也就是：
- 只有当 Adaptive Rail 进入 `CONFIRMED`
- 且当前主导轨道明确
- 且不是 `rail_broken`

才允许别的 4H 做多逻辑开仓。

这个方向的价值在于：
- 它不一定提高单策略绝对收益
- 但很可能能减少其他做多策略在结构不干净时乱开单

如果后面要继续扩展，我最看好它往这个方向走。

#### 4. 作为“实盘前的小仓试投策略”

如果后面进到实盘，我不会建议它第一天就承担主仓位。

我会建议的上线方式是：

1. 先只上 `BTC 4H`
2. 只上默认底座
3. 单策略小仓试投
4. 先观察：
   - 主导轨道是否频繁切换
   - `rail_broken` 平仓是否过多
   - 实盘成交后和回测的滑点差异

也就是说，它更适合：

```text
先作为低风险试投策略上线，
验证“结构识别在真实成交里是否还成立”
```

### 我不准备怎么用

为了避免方向跑偏，这几种用法我现在不建议：

#### 1. 不建议直接扩到 `1H`

当前研究已经很明确：
- `1H` 没有稳定优势
- 噪声太大
- 很容易把这套结构逻辑做坏

所以现在继续把它往 `1H` 扩，
大概率是在消耗时间，不是在提高价值。

#### 2. 不建议马上扩到多币种

现在这套结论是：
- 基于 `BTC`
- 基于 `4H`
- 基于趋势结构明显的市场

如果直接扩到很多币种，问题会立刻变成：
- 有些币根本不尊重这种中段轨道
- 有些币波动形态和 BTC 完全不同

所以顺序应该是：

```text
先把 BTC 4H 做扎实，
再考虑挑少数高流动币做迁移验证
```

#### 3. 不建议现在就当“唯一主策略”

虽然它现在已经很像质量最干净的版本，
但绝对收益还不是最强。

所以更成熟的做法是：
- 把它放进“主研究池”
- 把它当成默认底座继续打磨
- 后面考虑和其他 4H 做多策略分工

而不是现在就把所有资源只压它一条。

### 当前我对它的最终定位

如果你要我用一句话定义它：

```text
Adaptive Rail 现在最适合被用成：
BTC 4H 的结构型、低回撤、质量优先做多策略，
并且很适合作为后续其他做多策略的结构过滤层。
```

### 我建议的下一步使用路径

如果接下来进入“从研究走向可用”，我建议顺序是：

1. 继续把它作为默认 4H 回测底座
2. 增加“结构过滤器模式”研究
3. 研究它和 `EMA Breakout 4H` / `EMA Dynamic 5/13 4H` 的组合分工

也就是说，如果后面要留：

> 它更适合被定义成一个可选的 `4H 低回撤过滤层`，而不是当前主线策略的默认必经门槛。
