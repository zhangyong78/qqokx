# 划线交易验收清单

本文档汇总「划线交易台」与关联能力的产品规则与验收项，供实现后逐条核对。

---

## 一、API 与页面基础

- [ ] 划线交易台界面**展示当前 API / 环境**（与现有账户或 key 别名一致，可读即可）。
- [ ] 支持**切换 API**；切换后行情、持仓、下单通道与该 API 一致。
- [ ] **切换 API 时自动清空当前页面上已画的线**（趋势线、水平线、止损线、盈亏比划线等），避免误用旧线在新账户上触发。

---

## 二、趋势线下单（射线 + 穿越触发）

### 2.1 线型与几何

- [ ] 仅支持**射线**：由两点确定，**向右无限延伸**；可为水平（斜率为 0）或斜线。
- [ ] 新 K 线到达后，同一条射线的**延长**与几何一致（锚点不变，仅向右延伸）。

### 2.2 穿越定义（可配置）

- [ ] 用户可在**小卡片 / 集中参数区**选择穿越判定：
  - **Tick**：以**最新价**相对射线判定穿越；
  - **收盘**：以当前 K 线**收盘价**相对射线判定穿越。
- [ ] 文档或 UI 上简要说明两种口径的差异，避免误用。

### 2.3 触发与生命周期

- [ ] **触发频率**：**一次性**——触发执行配置动作（下单 / 站内通知等）后，该条划线策略**立即失效**，不再重复触发。
- [ ] 支持**有效期**（若已实现）：过期未触发则自动失效，与「一次性」语义一致时可合并验收。

### 2.4 动作与通知

- [ ] 穿越后可配置：**自动委托**（方向、数量规则见第四节、第五节）与/或**通知**。
- [ ] **邮件**：与**站内通知同一套通道/文案策略**（不单独维护另一套邮件产品逻辑，除非站内本身走邮件）。

---

## 三、盈亏比划线（独立功能，偏挂委托）

> 与「趋势线下单」射线为**不同**的划线能力：用于**风险回报可视化 + 挂委托单**场景，单量仍按**以损定量**。

- [ ] 提供类似参考图（多头/空头风险区）的**盈亏比划线工具**：含进场价、止损区间、目标区间、**盈亏比**展示。
- [ ] **计算基准**：以**委托单的开仓价 / 挂单成交价**为基准计算止盈、止损距离与**数量**（而非仅用图表上随意拖拽的瞬时价作为最终成交假设时，需在提交委托时以**实际挂单价格**重算或锁定逻辑一致）。
- [ ] 与「以损定量」一致：**账户规模或风险金、止损距离**决定数量；**盈亏比**决定止盈相对止损的比例或目标价。
- [ ] 提交后为**委托单**路径（限价/计划委托等以现有交易所能力为准），验收时核对委托参数与划线面板一致。

---

## 四、开多 / 开空按钮（市价类入口）

> 指工具栏上**开多 / 开空**按钮行为；参数与页面其它模块尽量**共用同一张参数小卡片**。

- [ ] 止损参考：**当前价**与**上一根 K 线**的高/低结合 **ATR（可设倍数）**（具体公式与现有「最新价 + 上一根 ATR」等选项对齐或统一命名）。
- [ ] **ATR 周期**：可配置，**默认 10**。
- [ ] **风险金**（或风险占账户比例，与产品统一）：可配置；数量由**止损距离 + 风险金**推导（以损定量）。
- [ ] **K 线周期、止损基准选项（如上一根 / 前三根 / 止损线等）**：若页面多处用到，均在**同一张集中参数小卡片**中可改，避免多处不一致。

---

## 五、集中参数小卡片（全页共用）

- [ ] 存在**统一参数区**（小卡片/侧栏）：至少包含 **ATR 周期（默认 10）、ATR 倍数、风险金/风险比例、穿越判定（Tick/收盘）、K 线周期或与图表联动说明** 等；开多开空、趋势线下单、盈亏比挂单从该处读取默认或可覆盖策略以文档为准。
- [ ] 修改卡片参数后，**未提交的划线**与**新开仓预览数量**随之更新（验收时抽查一两项即可）。

---

## 六、K 线与图表交互

- [ ] **K 线实时更新**（推送或轮询与项目一致）。
- [ ] 历史加载目标：**约 1000 根** K 线仍可流畅操作（允许分步加载，但视觉上应能浏览到足够深度）。
- [ ] 划线交易台主界面：**左右拖动、缩放**可用。
- [ ] **从工作台激活的「仅实时 K 线」视图**：不要求拖放缩放（轻量即可），与主交易台区分验收。

---

## 七、策略与工作台 / 首页

- [ ] 已配置且处于运行状态的趋势线触发策略，可在**首页「运行中策略」**或等价入口中查看与管理（启停、查看最近触发时间等，与项目现有策略列表能力对齐）。
- [ ] **工作台**中可与普通策略采用**同类管理模式**（列表、状态、删除/停用）。
- [ ] 历史成交/历史仓位中可见划线相关成交，**不破坏**原有普通策略统计（与旧清单第 10 条一致）。

---

## 八、回归场景（原清单操作流，保留作冒烟）

- [ ] 打开任一运行中会话的实时 K 线图，确认出现「画线工具 + 划线交易」操作栏。
- [ ] 依次选择「趋势线 / 水平线 / 止损线」在图上拖拽，确认刷新后线条仍可见（**注意**：趋势线在本文档中定义为**射线**，若 UI 仍称「趋势线」则几何须为射线）。
- [ ] 使用卡片中的默认/测试参数，点击「开多/开空」，确认日志出现划线交易下单记录。
- [ ] 切换「前三根高低」等止损基准（若仍保留），确认止损计算口径变化且可提交。
- [ ] 选择「止损线」基准时，先画止损线再下单，确认提交成功且日志包含入场价/止损价/数量。
- [ ] 「对手价」等下单模式若保留，确认可提交并进入成交追踪状态。
- [ ] 成交后若存在「启动动态管理」，确认后续止盈止损日志按现有动态委托风格输出。
