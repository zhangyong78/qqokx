待审核方案 / 参数区紧凑化

把“策略参数 + 回测参数”收成一屏主操作区

这版不只是压缩“回测参数”，而是把上半区整体重排。核心是按使用频率和修改习惯分层： 高频参数留在第一视线，低频开关收进胶囊和短说明，动态保护规则保持独立但更紧凑， 尽量减少整块区域的纵向占用，避免一打开就先看到滚动条。

  • 第一层放常改项： 策略、交易对、K线周期、均线、ATR、止盈、风险资金、时间区间、K线数。
  • 第二层放低频项： 复利、纯本地回测、时间保本、手续费偏移、同步周期等收为短标签或工具条。
  • 规则区独立： 动态保护规则不再拉长主表单，但保持可见，避免藏得太深。
  • 说明改短句： 长提示改成底部提示条，只保留决策需要的信息，不再占一整行解释文。
回测参数区紧凑版示意 目标：1920 宽度下尽量不出现顶部滚动条，且保留现有功能口径
第一屏优先看什么 先看到“策略怎么跑、风险怎么定、时间怎么切”，不是先看到大段说明文字和稀疏空白。
哪些保留展开 动态保护规则仍然直接可见，因为它是策略行为的一部分，不建议彻底折叠隐藏。
哪些改成短控件 复利、纯本地、时间保本、手续费偏移、同步周期这类低频项，统一改成紧凑标签行。

这版重点

  • “策略参数”和“回测参数”统一放进一屏主操作区，不再上下两块松散堆叠。
  • 高频字段采用两列或三列密排，低频状态用胶囊表达，减少空白。
  • 规则表保持可见，但从“拖长页面的主因”改成“高度可控的独立卡片”。

落地建议

  • 正式 UI 可保留当前字段和变量，不改业务逻辑，只先调整容器分组和 grid 布局。
  • 长提示文统一改成底部提示条或分组脚注，减少每一行字段后面的说明文本。
  • 参数区和回测报告区继续保留上下拖动分隔，用户能自行取舍可视高度。
