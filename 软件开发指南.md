# 软件开发指南

当前版本：`v0.5.07`

这份文档用于后续维护 `qqokx` 项目，目标是让人快速知道：

- 当前软件都有哪些主要功能
- 某类需求应该改哪个文件
- 本地数据和缓存放在哪里
- 回测、报告、打包分别走哪条链路
- 发布时哪些文件需要同步更新
- 长期协作时线程应该如何拆分

## 0. 里程碑版本

当前里程碑版本：`v0.5.07`

这一版重点在划线交易与数据联动，核心是：

- 策略实盘划线交易、触发算法单与图表配套
- 历史成交与订单/持仓视图协同刷新体验
- EMA 金叉死叉洞察脚本与回测、客户端配套扩展
- 发版版本号、更新日志与打包产物继续保持一致

如果后面继续做功能改动，建议先看这几份文件：

- `README.md`
- `发版协作约定.md`
- `发版待打包清单.md`
- `线程工作流模板.md`

## 7. 打包、发布与版本同步

### 7.1 打包脚本

使用：

```powershell
python D:\qqokx\scripts\build_server_package.py --version 20260415_001556
```

### 7.2 每次代码功能变更后要同步的文件

- `pyproject.toml`
- `okx_quant/__init__.py`
- `README.md`
- `软件开发指南.md`

如果采用多线程协作，还要同步检查：

- `发版待打包清单.md`
- `发版协作约定.md`

### 7.3 当前协作规则

- 功能线程里用户说 `123`
  - 默认表示：
    - 根据当前线程主题和当前 diff
    - 把已经完成、准备进入下一版的内容登记到 `发版待打包清单.md`
  - 默认不直接打包
  - 默认不直接提交和推送
- `总控-版本路线图` 或 `数据-缓存-导出-发版` 线程里用户说 `123`
  - 默认表示：
    - 读取 `发版待打包清单.md`
    - 判断这次发版的版本号
    - 更新 `pyproject.toml`
    - 更新 `okx_quant/__init__.py`
    - 更新 `README.md`
    - 打包
    - 提交
    - 推送
- 打包前要尽量完成：
  - 版本号同步
  - 关键测试通过
  - 发布说明补齐
- 如果工作区存在无关脏改：
  - 先排除无关文件
  - 不把临时报告、缓存和调试产物误带进版本
- 具体规则见：
  - `发版协作约定.md`

### 7.4 版本递增规则

- `patch`
  - 修文档
  - 修提示文案
  - 修 UI 小问题
  - 修测试
  - 修 bug
  - 不改变主要功能结构
- `minor`
  - 新增一个明显的新功能
  - 新增一个新窗口、新面板、新工作流
  - 原有功能能力明显增强
  - 但没有把整条产品线推翻重来
- `major`
  - 版本线切换
  - 架构大改
  - 数据目录、发版链路、线程协作规则这种底层约定大重构
  - 影响面很大，后续兼容要专门处理

版本判断原则：

- 以“用户可感知的最大变化”作为版本级别判断依据
- 如果同一版里既有小修小补，又有大功能，优先按影响最大的变化判断
- 如果只是记录、整理、润色和文档收口，通常按 `patch`

## 9. 维护建议

- 先定位需求属于哪一块，再动对应文件，不要在 UI 里硬改核心逻辑
- 回测、实盘、导出三条链路改动时，尽量一起检查
- 涉及版本变化时，优先保证：
  - 软件标题版本一致
  - README 版本一致
  - 打包文件名版本一致
- 每次生成长期有价值的回测总结时，优先放到 `reports/` 根目录，并补到 `reports/回测任务索引.md`

## 10. 数据目录与恢复接管

统一数据目录：
- 入口在 `main.py` 与 `okx_quant/app_paths.py`
- 默认数据目录为代码目录同级的 `qqokx_data/`
- 可通过 `--data-dir` 或环境变量 `QQOKX_DATA_DIR` 覆盖

目录分层：
- `config/`：凭证、通知设置、增强策略运行配置
- `cache/`：K 线缓存、Deribit 波动率缓存
- `state/`：回测历史、策略历史、Smart Order 任务、可恢复会话注册表
- `logs/`：系统日志、策略独立日志
- `reports/`：回测导出、分析导出、Deribit 导出、实盘会话目录

关键路径入口：
- `okx_quant/persistence.py`：统一持久化路径与快照读写
- `okx_quant/log_utils.py`：统一日志目录
- `okx_quant/deribit_volatility_ui.py`：Deribit 缓存与导出走数据目录
- `okx_quant/option_strategy_ui.py`：Deribit 缓存读取走数据目录
- `okx_quant/enhanced_live_engine.py`：增强策略状态快照、事件流、会话目录
- `okx_quant/ui.py`：待恢复会话注册、自动恢复、手动恢复入口

恢复边界：
- 已支持：现货增强三十六策略的保守恢复接管
- 已有持久化：Smart Order 任务恢复
- 暂未承诺：普通策略自动接管

打包与升级：
- 打包脚本默认不携带运行期数据目录
- 升级时优先保留原 `qqokx_data/`，仅替换代码目录
- 若迁移机器，可整体复制 `qqokx_data/` 后再启动新版本
- 文档与打包说明需同步说明 `--data-dir` / `QQOKX_DATA_DIR`

Git 提交建议补充：
- 不提交 `qqokx_data/` 下的运行期数据
- 不提交缓存、导出、临时分析产物和打包输出
