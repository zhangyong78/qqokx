# 软件开发指南

当前版本：`v0.6.19`

这份文档用于后续维护 `qqokx` 项目，目标是让人快速知道：

- 当前软件都有哪些主要功能
- 某类需求应该改哪个文件
- 本地数据和缓存放在哪里
- 回测、报告、打包分别走哪条链路
- 发布时哪些文件需要同步更新
- 长期协作时线程应该如何拆分

## 0.1 协作快捷口令

为减少来回确认，当前长期协作约定如下：

- 你说 `123`，就表示要同步更新 `README.md` 和 `软件开发指南.md`
- 默认场景是：把当前已完成功能、目录变化、流程规则变化同步写入这两份文档
- 如果只更新其中一份，或者还要连带更新别的文档，需要在口令外额外说明

后续维护时，把 `123` 视为固定文档同步指令即可。

## 0. 里程碑版本

当前里程碑版本：`v0.6.19`

这一版的重点已经变成“策略系统升级 + 新策略离场细化 + 回测研究补齐”，核心包括：

- 新增 `EMA55 斜率做空` 策略：
  - 策略文件：`okx_quant/strategies/ema55_slope_short.py`
  - 开空条件：`EMA55` 单根斜率比例小于等于阈值
  - 平仓条件：现在支持显式配置是否启用“斜率重新转正平仓”
  - 支持 launcher / backtest 双端参数
- 这条策略的动态保护参数已经进一步统一：
  - 新增 `ema55_slope_lock_profit_trigger_r`
  - `dynamic_two_r_break_even` 在这条策略上解释为“首档保本特例”
  - 当 `首档触发R = 2` 时，可先抬到保本位；否则首档按 `N-1R` 逻辑上移
  - 启动确认、会话恢复、回测参数摘要、导出报告都已按同一口径展示
- `EMA55 斜率做空` 的通用版参数默认值更新：
  - 在 `strategy_parameters.py` 中固定 `ema_type=ema`、`ema_period=55`
  - 在 `strategy_ui_schema.py` 与 `strategy_symbol_defaults.py` 中默认开启 `ema55_slope_exit_enabled`
  - 默认 `ema55_slope_lock_profit_trigger_r = 2`
- 这条策略的回测与风险管理口径已经补齐：
  - 固定风险金 / 固定数量 / 风险百分比
  - `2R 保本`
  - 手续费偏移
  - 时间保本
  - 动态止盈
- 新增 `strategy_ui_schema.py`
  - 集中管理策略 UI 默认值
  - 集中管理控件显示隐藏
  - 集中管理强制跟随信号 / 强制本地止盈止损 / 固定额外值
- 新增 `strategy_runtime_registry.py`
  - 集中管理策略 family
  - 集中管理 signal / local / exchange handler
  - 集中管理方向偏好、参考线标题、dynamic / mtf / signal extrema helper
- `engine.py`、`engine_strategy_router.py`、`backtest.py`、`backtest_ui.py`、`ui_shell.py`、`ui_strategy_sessions.py`
  已开始共用 schema / registry，减少新增策略时对主程序初始化和主流程的侵入式修改
- 本轮还修复了“新增策略导致主程序启动即退出”的问题，并补了启动烟测、参数恢复、策略切换和运行路由回归
- 本轮继续补了这条策略的参数恢复与回填链路：
  - `ui_shell.py`：启动区新增平仓条件和 `首档触发R`
  - `ui_strategy_sessions.py`：会话详情、编辑回填、快照摘要同步支持新参数
  - `backtest_ui.py`：回测区新增对应控件、可见性控制、摘要说明
  - `backtest.py` / `backtest_export.py`：回测报告与参数摘要按新口径导出
- 另外新增了一批研究脚本与报告产物：
  - `S089` 日线 EMA 对比
  - `S096/S097` 距离确认、最小距离窗口、远距入场提前保护、亏损形态、保护后再入场

另外，下面这些能力依然是当前现行能力，后续维护时要一起考虑：

- `现货套利` 整体增强：
  - 扫描支持 `永续 / 交割` 类型过滤、`币种` 过滤、`净年化 / 基差 / 到期天数` 排序、`绝对价差` 列
  - 套利开仓 / 套利平仓默认使用 `绝对价差`
  - 新增 `持仓配对平仓`，可直接基于账户真实现货与交割/永续持仓做配对平仓
  - 新增 `交割合约移仓`，可基于已有交割合约套利持仓滚动到更远交割合约，现货腿保持不动
  - 支持部分平仓、分批执行、挂单腿 / 吃单腿、追单次数
  - `套利开仓 / 套利平仓 / 持仓配对平仓 / 交割移仓` 都有双腿盘口 / 价差面板
  - 双盘口优先走 `OKX 公共 WS`，失败时自动回退 `REST`
  - 套利开仓 / 套利平仓也支持 `双腿吃单 / 现货挂单合约吃单 / 合约挂单现货吃单`
- `OKX 私有 WebSocket 加速层` 已落地：
  - 新文件 `okx_quant/okx_private_ws.py`
  - `orders / positions / account` 已接入私有 WS 缓存
  - 成交回报与持仓状态优先使用 WS，失败时自动回退 REST
  - 开关环境变量：`QQOKX_PRIVATE_WS_ENABLED`
- `OKX 公共 WebSocket 行情层` 已落地：
  - 新文件 `okx_quant/okx_public_ws.py`
  - 当前先服务本地现货套利窗口双盘口
  - 开关环境变量：`QQOKX_PUBLIC_WS_ENABLED`
- `策略实盘扩容优化` 已补齐第一阶段：
  - 默认 `poll_seconds` 已从 `3s` 调整为 `10s`
  - 新增 `okx_quant/market_data_hub.py`，同进程内相同 `instId + bar` 的 K 线共享读取
  - 触发价优先读取 `公共 WS ticker`
  - 订单状态优先读取 `私有 WS orders`
  - 拿不到缓存或缓存未更新时自动回退 `REST`
- `持仓 / 会话 UI` 已补充 WS 缓存状态提示：
  - 持仓摘要会显示 `持仓WS vN / 账户WS vN` 或“已回退 REST”
  - 运行中策略汇总、单条策略详情会显示当前持仓缓存说明

### 2.4.1 当前策略接入约定

后面继续新增策略时，优先按下面的链路接入，不要再直接往主程序堆 `if strategy_id == ...`：

1. 在 `strategy_catalog.py` 注册策略定义
2. 在 `strategy_parameters.py` 声明可用参数和固定参数
3. 在 `strategy_ui_schema.py` 声明：
   - launcher / backtest 默认值
   - 额外默认值
   - 固定额外值
   - 是否支持动态止盈
   - 是否强制跟随信号
   - 是否强制本地止盈止损
4. 在 `strategy_runtime_registry.py` 声明：
   - strategy family
   - signal / local / exchange handler
   - 方向偏好
   - 参考线标题 helper
   - 是否使用 dynamic orders / mtf filter / signal extrema
5. 最后实现 `okx_quant/strategies/` 下的信号逻辑，并补回测和路由测试
6. 如果策略 family 需要特殊动态保护文案或特殊首档止盈规则，优先补到：
   - `ui_shell.py` 的启动提示
   - `backtest_ui.py` 的参数摘要与显隐
   - `backtest.py` / `backtest_export.py` 的报告说明
   - `ui_strategy_sessions.py` 的快照展示与编辑回填

当前维护原则是：

- 新策略优先改 `catalog / parameters / ui_schema / runtime_registry / strategies`
- 没有必要时，不要直接改主程序初始化流程
- 如果新增策略又要求改 `ui_shell.py` 或 `engine.py` 的主流程分支，优先先想能不能把规则下沉到 schema / registry
- 如果新增的是策略特有保护参数，要同时检查“启动区 / 会话编辑 / 回测区 / 报告导出 / 测试”五条链路是否一起补齐

  - `_render_line_trading_desk_chart_impl`：`render_strategy_live_chart` 与 `compute_strategy_live_chart_layout` 共用同一套 `to_draw` / 锚点；RR 绘制与命中测试；`_line_trading_desk_draw_ray_on_canvas` 中趋势线用 `layout_price_to_y_unclamped` 与未钳位虚线延伸
  - `_line_trading_desk_desk_anchor_prices`：射线等可按 OHLC 带宽过滤；**盈亏比开仓/止损/止盈三档价须始终纳入**纵轴，避免矩形被钳成零高度（「框没了」）
  - `_line_trading_desk_rr_layout_rel_mid_clamped` 等：可见切片含右侧平盘 ghost K 时，用 `series_plot_end_index` 把 RR 的横向锚点钳在最后一根**真实** K，避免画进空白区
- `okx_quant/strategy_live_chart.py`
  - `strategy_live_chart_canvas_layout` / `render_strategy_live_chart` / `measure_strategy_live_chart_canvas`：**布局宽度/高度以已映射的 `winfo` 为准**（达到 `CHART_CANVAS_MIN_MAPPED_`* 才认为 `ready`），勿用 `cget("width")` 的创建默认值冒充当前像素，否则首帧 candle_step 与纵轴会整屏错位；`layout_price_to_y_unclamped` 供趋势线保持斜率
  - `slice_strategy_live_chart_snapshot_with_desk_right_pad`：占位 K 与 `series_plot_end_index`（均线截断与 RR 横向钳位共用语义）

**维护备忘（曾与「画线脱离 K 线」「闪一下又正常」相关）：**

1. **避免同一轮打开/刷新内两次切换 K 线总根数**（例如先 280 再 500）：用户若在中间态画线，绝对 `bar`_* 会越界或纵轴在两帧间跳变。现划线台 worker **只请求一次** `LINE_TRADING_DESK_CANDLE_TARGET`；若将来再优化「首包更快」，也应保证 UI **只认最终快照**或必须重映射锚点。
2. **画布未映射完成时不要画 K 线**：`render` 在 `not layout_ready` 时 `return False` 且不 `delete("all")`；desk 侧用短间隔重试链 + `<Map>` 直到 `strategy_live_chart_canvas_layout` 为 ready。
3. **RR 与占位 K**：切片右侧 ghost 仅用于留白，**几何与交互的「最后一根 K」**以 `series_plot_end_index` 为界。
4. **根数仍可能变化的场景**（换标的、换周期、手动刷新）：保留 `_line_trading_desk_should_remap_annotations_for_candle_change` + `_line_trading_desk_remap_annotations_bars_after_candle_reload`（按时间戳找新索引）。
5. **趋势线（`kind == "line"`）**：锚点与虚线延伸用 `layout_price_to_y_unclamped`，延伸段勿对 `y2` 做上下界钳位。若用 `layout_price_to_y`（ratio 钳在 0～1），界外价会被压到同一边导致像素上 `dy=0`，出现「折一下再水平拖出去」。X 向由 `bar_a` / `bar_b`（绝对 K 索引）减 `view_start` 换算，与横移视图一致。

- `patch`
  - 修文档
  - 修提示文案
  - 修 UI 小问题
  - 修测试
  - 修 bug
  - 不改变主要功能结构

## 10. 数据目录与恢复接管

统一数据目录：

- 入口在 `main.py` 与 `okx_quant/app_paths.py`
- 默认数据目录为代码目录同级的 `qqokx_data/`
- 可通过 `--data-dir` 或环境变量 `QQOKX_DATA_DIR` 覆盖

目录分层：

- `config/`：凭证、通知设置、增强策略运行配置
- `cache/`：K 线缓存、Deribit 波动率缓存
- `state/`：回测历史、策略历史、Smart Order 任务、可恢复会话注册表
- `logs/`：系统日志、策略独立日志
- `reports/`：回测导出、分析导出、Deribit 导出、实盘会话目录

关键路径入口：

- `okx_quant/persistence.py`：统一持久化路径与快照读写
- `okx_quant/log_utils.py`：统一日志目录
- `okx_quant/deribit_volatility_ui.py`：Deribit 缓存与导出走数据目录
- `okx_quant/option_strategy_ui.py`：Deribit 缓存读取走数据目录
- `okx_quant/enhanced_live_engine.py`：增强策略状态快照、事件流、会话目录
- `okx_quant/ui.py`：待恢复会话注册、自动恢复、手动恢复入口

恢复边界：

- 已支持：程序关闭或升级重启后，空闲中的普通策略会话可按注册表自动恢复
- 已支持：`等待信号` / `signal_only` 会话在未持仓时自动重启监听
- 已支持：`trade + exchange` 模式下，已有持仓或未完成挂单的会话继续走保守恢复接管
- 已有持久化：Smart Order 任务恢复
- 安全边界：非 `trade + exchange` 模式如果处于持仓中或挂单中，不强行自动接管
- 暂未承诺：所有普通策略的持仓中自动接管

打包与升级：

- 打包脚本默认不携带运行期数据目录
- 升级时优先保留原 `qqokx_data/`，仅替换代码目录
- 若迁移机器，可整体复制 `qqokx_data/` 后再启动新版本
- 文档与打包说明需同步说明 `--data-dir` / `QQOKX_DATA_DIR`

Git 提交建议补充：

- 不提交 `qqokx_data/` 下的运行期数据
- 不提交缓存、导出、临时分析产物和打包输出
