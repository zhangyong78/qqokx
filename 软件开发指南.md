# 软件开发指南

本文档用于帮助后续维护 `qqokx` 项目，快速定位：

- 文件和文件夹负责什么
- 某类需求应该改哪个文件
- 对应测试文件在哪里
- 本地运行文件是什么用途
- 迁移机器时哪些文件必须保留
- 哪些文件绝对不要上传到 GitHub

## 1. 项目主目录

项目根目录：

- `main.py`：程序入口
- `pyproject.toml`：项目版本号与元信息
- `requirements.txt`：依赖列表
- `README.md`：用户说明
- `软件开发指南.md`：开发维护说明
- `okx_quant/`：主程序源码
- `scripts/`：打包与批量脚本
- `tests/`：测试代码
- `reports/`：导出报告
- `dist/`：服务器打包产物

## 2. 核心源码模块

### 2.1 主界面与窗口

- `okx_quant/ui.py`
  - 主界面
  - 菜单
  - 账户持仓区
  - 持仓大窗
  - 各类主弹窗入口

- `okx_quant/window_layout.py`
  - 所有窗口自适应分辨率布局

### 2.2 OKX 执行与接口

- `okx_quant/engine.py`
  - 实盘策略执行引擎
  - 策略启动、监控、下单、止盈止损衔接

- `okx_quant/okx_client.py`
  - OKX REST 接口
  - 下单、查单、查仓、盘口、ticker、历史数据

- `okx_quant/models.py`
  - 主要数据结构定义

- `okx_quant/pricing.py`
  - 价格格式化
  - 精度处理
  - 最小变动单位相关逻辑

- `okx_quant/notifications.py`
  - 邮件通知逻辑

### 2.3 策略模块

目录：`okx_quant/strategies/`

- `ema_dynamic.py`
  - EMA 动态委托策略

- `ema_atr.py`
  - EMA ATR / 穿越市价策略

- `ema_cross_ema_stop.py`
  - 4H EMA5/EMA8 金叉死叉策略

- `okx_quant/strategy_catalog.py`
  - 策略注册表和显示名称

### 2.4 回测系统

- `okx_quant/backtest.py`
  - 回测核心逻辑
  - 手续费、滑点、资金曲线、矩阵批量回测

- `okx_quant/backtest_ui.py`
  - 回测窗口
  - 当前报告 / 回测对比 / 矩阵对比 / 热力图
  - 图表缩放拖动

- `okx_quant/backtest_export.py`
  - 回测报告导出

- `okx_quant/candle_cache.py`
  - 本地 K 线缓存

### 2.5 期权持仓保护

- `okx_quant/position_protection.py`
  - 期权保护核心逻辑
  - 网络重试
  - 价格保护
  - 防重复平仓
  - 反向仓位熔断

- `okx_quant/protection_replay_ui.py`
  - 保护回放模拟窗口

### 2.6 多币种信号监控

- `okx_quant/signal_monitor.py`
  - 信号判定逻辑

- `okx_quant/signal_monitor_ui.py`
  - 信号监控窗口
  - 任务、日志、诊断区

### 2.7 Deribit 波动率模块

- `okx_quant/deribit_client.py`
  - Deribit 接口封装

- `okx_quant/deribit_volatility_ui.py`
  - Deribit 波动率指数 K 线工具

- `okx_quant/deribit_volatility_monitor.py`
  - 波动率信号监控逻辑

- `okx_quant/deribit_volatility_monitor_ui.py`
  - 波动率监控窗口

### 2.8 无限下单 / Smart Order

- `okx_quant/smart_order.py`
  - 无限下单核心
  - 条件单
  - 网格任务
  - 总仓位限制
  - 任务恢复

- `okx_quant/smart_order_ui.py`
  - 无限下单窗口
  - 规则盘口
  - 自选合约
  - 条件单、网格、仓位限制界面

## 5. 本地 JSON / 本地目录说明

### 5.1 主要本地文件

- `.okx_quant_credentials.json`
  - OKX API 凭证
  - 多 API 配置

- `.okx_quant_settings.json`
  - 邮件设置
  - 环境
  - 持仓模式
  - 交易模式

- `.okx_quant_backtest_history.json`
  - 回测历史

- `.okx_quant_smart_order_tasks.json`
  - 无限下单任务快照

- `.okx_quant_smart_order_favorites.json`
  - 无限下单自选合约

### 5.2 本地目录

- `.okx_quant_candle_cache/`
  - 本地 K 线缓存

- `reports/`
  - 回测汇总
  - 推荐清单
  - 详细导出

- `dist/`
  - 服务器打包产物

## 6. 更新服务器时建议保留

### 6.1 必须保留

- `.okx_quant_credentials.json`
- `.okx_quant_settings.json`

### 6.2 建议保留

- `.okx_quant_backtest_history.json`
- `.okx_quant_smart_order_tasks.json`
- `.okx_quant_smart_order_favorites.json`
- `.okx_quant_candle_cache/`

### 6.3 按需保留

- `reports/`

## 7. 不要上传到 GitHub 的文件

### 7.1 绝对不要上传

- `.okx_quant_credentials.json`
- `.okx_quant_settings.json`
- `.env`
- `.env.*`
- `.okx_quant_smart_order_tasks.json`
- `.okx_quant_smart_order_favorites.json`

### 7.2 建议不要上传

- `.okx_quant_candle_cache/`
- `dist/`
- `debug_*.txt`
- `reports/` 中带有个人策略结论和运行结果的文件

## 8. 迁移机器时最短清单

### 8.1 只想保住“能继续用”

- `.okx_quant_credentials.json`
- `.okx_quant_settings.json`

### 8.2 想保住“交易连续性”

再加：

- `.okx_quant_smart_order_tasks.json`
- `.okx_quant_smart_order_favorites.json`

### 8.3 想保住“分析连续性”

再加：

- `.okx_quant_backtest_history.json`
- `.okx_quant_candle_cache/`

## 9. 打包相关

- `main.py`

- `README.md`
- `软件开发指南.md`
- `okx_quant/`

## 10. 维护建议

### 10.1 改功能时先找对应测试

建议流程：

1. 先找功能对应代码文件
2. 再找对应测试文件
3. 修改后先跑专项测试
4. 再跑全量测试

### 10.2 新增功能时

建议同步做：

- UI 说明
- README 更新
- 必要测试
- 是否需要本地持久化评估
- 是否需要打包脚本纳入评估
