# 回测要求标准文件 v1

以后涉及 EMA突破做多 / EMA跌破做空 策略回测, 默认先读取本文件。除非用户明确修改, 否则按本文件执行、出报告和归档。

## 一、基础目标

- 目的: 验证策略是否在真实交易成本后仍有优势。
- 不能只看单一收益最高组合, 必须看稳定性、周期一致性、压力滑点、长样本和是否过拟合。
- 新策略必须多空分开回测, 不把做多和做空混在同一个结果里。
- 同一策略必须区分“主矩阵筛选”和“二轮确认”。

## 二、默认交易标的

默认五币:

- `BTC-USDT-SWAP`
- `ETH-USDT-SWAP`
- `SOL-USDT-SWAP`
- `DOGE-USDT-SWAP`
- `BNB-USDT-SWAP`

如新增币种, 必须在报告里明确写入新增币种和未跑币种。

## 三、默认策略范围

- `ema_breakout_long`: EMA突破做多, 只做多。
- `ema_breakdown_short`: EMA跌破做空, 只做空。

策略必须分别输出:

- 做多报告
- 做空报告
- 多空横向对照
- 币种横向对照
- 周期横向对照

## 四、默认周期

主矩阵周期:

- `15m`
- `1H`
- `4H`

判断规则:

- `1H/4H` 是主验证周期。
- `15m` 只作为观察周期, 不作为第一阶段实盘采用依据。
- 如果15m要进入二轮, 至少跑30000根。

## 五、默认样本长度

主矩阵:

- 每组 `10000` 根K线。
- 前置预热按系统要求自动处理。

二轮确认:

- `15m`: 至少 `30000` 根。
- `1H`: 尽量全量历史。
- `4H`: 尽量全量历史。

报告必须写明:

- 实际K线数量
- 数据起止时间
- 是否使用本地缓存
- 是否发生补拉

## 六、默认资金与成本

- 初始本金: `10000`
- 仓位模式: 固定风险金
- 单笔风险金: `10`
- 复利: 关闭
- Maker手续费: `0.015%`
- Taker手续费: `0.036%`
- 主滑点: 开仓 `0.03%`, 平仓 `0.03%`
- 压力滑点: 开仓 `0.05%`, 平仓 `0.05%`
- 资金费: `0`

说明:

- 突破/跌破属于直接下单策略, 不允许用0滑点作为正式报告口径。
- 如果临时跑0滑点, 只能标注为调试, 不能作为正式结论。

## 七、默认参数矩阵

基础指标:

- EMA周期: `21`
- 趋势EMA周期: `55`
- ATR周期: `10`
- 每波最多开仓次数: `1`

入场参考EMA:

- `21`
- `55`

止损ATR倍数:

- `1`
- `1.5`
- `2`

动态止盈主版本:

- `take_profit_mode = dynamic`
- `dynamic_two_r_break_even = true`
- `dynamic_fee_offset_enabled = true`
- `time_stop_break_even_enabled = false`
- `hold_close_exit_bars = 0`
- ATR止盈参数默认矩阵: `2 / 3 / 4`

固定止盈对照版本:

- `take_profit_mode = fixed`
- `dynamic_two_r_break_even = false`
- `dynamic_fee_offset_enabled = false`
- `time_stop_break_even_enabled = false`
- `hold_close_exit_bars = 0`
- TP必须按R倍数解释: `TP1R / TP2R / TP3R / TP4R`
- 固定止盈实际ATR距离 = `止损ATR倍数 * TP_R`

重要口径:

- `atr_take_multiplier=2` 是2ATR, 不一定等于2R。
- 只有 `SL=1ATR` 时, `atr_take_multiplier=2` 才等于2R。
- 固定止盈报告必须写明到底是ATR倍数还是R倍数。

## 八、执行顺序

第一步: 主矩阵筛选

- 五币
- 两策略
- 三周期
- 主滑点0.03%/0.03%
- 动态止盈主版本

第二步: 固定止盈对照

- 五币
- 两策略
- 三周期
- 主滑点0.03%/0.03%
- 固定止盈不保本
- TP按R倍数: 1/2/3/4

第三步: 二轮确认

- 只对A档和重点B档组合执行。
- 压力滑点0.05%/0.05%。
- 1H/4H全量历史。
- 15m至少30000根。

第四步: 稳定性拆分

- 按年度拆分。
- 按市场阶段拆分。
- 输出最大连续亏损。
- 输出最大单月回撤。
- 输出手续费+滑点占毛利比例。

## 九、报告要求

每轮回测至少输出:

- 人类可读Markdown报告
- 结构化JSON结果
- 每组胜出参数
- Top组合列表
- A/B/C优先级分层
- 币种维度结论
- 策略方向维度结论
- 周期维度结论
- 是否建议进入二轮验证

单策略正式报告还应包含:

- 交易数
- 胜率
- 总盈亏
- 收益率
- Profit Factor
- 平均R
- 最大回撤
- 手续费
- 滑点成本
- 止盈命中
- 止损命中
- 资金曲线
- 交易明细
- 审计JSON

## 十、结论用语规范

禁止直接把主矩阵结论写成“可上线”。

推荐用语:

- `主矩阵通过, 可进入二轮验证`
- `压力滑点通过, 可进入观察盘`
- `长样本通过, 可小资金跟踪`
- `15m仅观察, 暂不纳入第一阶段`
- `固定止盈为对照版本, 动态止盈为当前主版本`

## 十一、当前已知结论摘要

- 动态止盈 + 2R保本整体优于固定止盈不保本。
- ETH 1H EMA突破做多是当前最高优先级。
- DOGE/BNB做空在主矩阵里表现突出, 但需要压力滑点和长样本。
- BTC 4H做多/做空都有观察价值。
- SOL 1H做多/做空、4H做空可继续验证。
- 15m整体噪音较大, 不建议第一阶段实盘。
