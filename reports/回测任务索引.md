# 回测任务索引

这份索引用来解决两个问题：

- 以前让 Codex 跑过哪些批量回测，不容易从聊天列表里直接翻出来
- 左侧线程列表通常只显示“几天前 / 几周前”，不显示精确到分钟的时间

因此，最稳的定位方法是：

1. 先看这里的文件索引
2. 再看对应导出文件内容
3. 最后回聊天里按关键词搜，而不是按时间硬翻

## 1. 快速定位方法

### 1.1 优先搜关键词

推荐优先在聊天里搜这些词：

- `EMA动态委托`
- `浓缩表`
- `实盘推荐`
- `15分钟`
- `1小时 4小时`
- `10000根`
- `专业交易员建议`

### 1.2 再看本地历史快照

本地文件：

- [D:\qqokx\.okx_quant_backtest_history.json](/D:/qqokx/.okx_quant_backtest_history.json)

作用：

- 保存回测窗口里的历史快照
- 每条记录会带：
  - `snapshot_id`
  - `created_at`
  - `inst_id`
  - `bar`
  - `signal_mode`
  - `export_path`

注意：

- 这个文件更适合查“最近一段时间在回测窗口里跑过什么”
- `reports` 根目录里的总结文件，更适合查“你让我单独整理出来的阶段性结论”

## 3. 从文件反查聊天的建议顺序

如果你以后拿到一个历史文件，建议按这个顺序找：

1. 先看文件名
   - 周期
   - 币种
   - 是否是 `summary`
   - 是否是 `浓缩表`
   - 是否是 `实盘推荐`

2. 再看文件时间
   - 比如：
     - `2026-03-26 22:45 ~ 22:47`
     - `2026-03-27 07:22`

3. 最后去聊天里按主题词搜
   - 不建议只按“几周前”硬翻

## 4. 后续维护规则

以后如果再次出现这类“单独导出的阶段性回测总结”，建议同步做两件事：

1. 文件继续放在 `reports/` 根目录
2. 在这份 [回测任务索引.md](/D:/qqokx/reports/回测任务索引.md) 里补一条

建议每条至少写清楚：

- 文件名
- 生成时间
- 回测主题
- 适合搜索的聊天关键词
