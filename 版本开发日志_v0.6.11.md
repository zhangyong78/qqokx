# 版本开发日志 v0.6.11

更新时间：`2026-06-05`

## 这轮做了什么

这一轮版本主要完成了三件事：

1. 新增 `EMA55 斜率做空` 策略，并把实盘默认参数、回测参数、研究报告和相关测试补齐。
2. 修复了新策略接入引发的主程序启动崩溃问题。
3. 推进“策略与主程序解耦”的 B 方案，把策略 UI、运行路由、回测 family 和部分执行细节抽成统一 schema / registry。

## 功能更新

### 1. EMA55 斜率做空

新增文件：

- [okx_quant/strategies/ema55_slope_short.py](/D:/qqokx/okx_quant/strategies/ema55_slope_short.py)

核心规则：

- 当 `EMA55` 单根斜率比例小于等于阈值时开空
- 当斜率重新转正时平仓
- 斜率阈值可调，默认值为 `-0.0005`

配套更新：

- launcher / backtest 参数接入
- UI 提示文案补充
- 回测支持固定风险金、固定数量、风险百分比
- 回测支持 `2R 保本`、手续费偏移、时间保本、动态止盈

## 当前结论

这一轮版本已经从“继续堆新策略”转到“让新策略更安全地接入系统”。

当前可以把结构层定义为：

- 已达到“可持续新增策略”的基础状态
- 已不再需要每加一个策略都大范围改主程序
- 剩余耦合更多是业务差异，不再是架构性阻塞

## 后续建议

后面如果继续迭代，建议按这个优先级：

1. 优先做实盘参数、风控和研究质量
2. 只有在新增策略再次暴露新耦合时，再小步补 schema / registry
3. 暂不急着继续做更重的插件化拆分
